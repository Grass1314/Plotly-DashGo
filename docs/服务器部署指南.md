# DashGo 服务器部署指南（systemd 方式）

> 本文以 CentOS / Ubuntu 等主流 Linux 发行版为例，使用 systemd 管理 DashGo 主应用和任务调度两个进程。

---

## 前置条件

| 项目 | 要求 |
|------|------|
| 操作系统 | Linux（CentOS 7+、Ubuntu 18.04+、Debian 10+ 等） |
| Python | >= 3.9 |
| MySQL | >= 5.7（已安装并运行，已创建好数据库和用户） |
| 网络 | 服务器能访问 MySQL，且开放应用端口（默认 8090） |

---

## 第一步：上传项目代码

将项目代码上传到服务器，假设部署目录为 `/opt/dashgo`：

```bash
# 方式一：git clone
cd /opt
git clone https://你的仓库地址.git dashgo

# 方式二：本地打包上传
# 本地执行：
tar czf dashgo.tar.gz --exclude='.venv' --exclude='__pycache__' --exclude='.idea' --exclude='app.db' Plotly-DashGo/
scp dashgo.tar.gz root@你的服务器IP:/opt/

# 服务器执行：
cd /opt && tar xzf dashgo.tar.gz && mv Plotly-DashGo dashgo
```

**不需要上传的文件：**
- `.venv/` — 虚拟环境在服务器上重新创建
- `.idea/` — IDE 配置
- `app.db` — 本地 SQLite 数据库
- 任何 `.env` 文件 — 主密钥在服务器上单独创建，不通过代码仓库传递

**需要上传的文件：**
- `src/config/dashgo.ini`（含 `ENC(...)` 加密值）可以安全上传，因为没有主密钥无法解密

## 第二步：创建虚拟环境并安装依赖

```bash
cd /opt/dashgo

# 创建虚拟环境
python3 -m venv .venv

# 激活虚拟环境
source .venv/bin/activate

# 安装依赖
pip install -r requirements_linux.txt

# 验证安装（应无报错）
python -c "import dash; import peewee; import feffery_antd_components; print('依赖安装成功')"

# 退出虚拟环境
deactivate
```

---

## 第三步：在服务器上创建主密钥环境变量文件

这是整个部署中最关键的安全步骤。主密钥文件**只存在于服务器上**，不从本地上传。

```bash
# 创建配置目录
sudo mkdir -p /etc/dashgo

# 写入主密钥（替换为你的实际主密钥）
echo 'DASHGO_MASTER_KEY=DashGo@MasterKey2026!' | sudo tee /etc/dashgo/env

# 设置严格权限：仅 root 可读写
sudo chmod 600 /etc/dashgo/env
sudo chown root:root /etc/dashgo/env

# 验证权限（应显示 -rw------- root root）
ls -la /etc/dashgo/env
```

> 为什么不从本地上传 `.env` 文件？
> - 主密钥一旦暴露，所有加密配置等于明文，所以它不应该存在于代码仓库、本地项目目录、传输记录中
> - 在服务器上直接创建，通过 `chmod 600` 限制只有 root 能读取，systemd 以 root 权限读取后传递给应用进程
> - 如果你需要在多台服务器部署，在每台服务器上分别创建这个文件即可

## 第四步：初始化数据库

```bash
cd /opt/dashgo/src

# 临时加载主密钥（仅此终端会话有效）
export DASHGO_MASTER_KEY="DashGo@MasterKey2026!"

# 使用项目虚拟环境的 Python
PYTHON=/opt/dashgo/.venv/bin/python

# 创建所有数据库表（已存在则跳过）
$PYTHON -c "from database.sql_db.conn import create_rds_table; create_rds_table()"
echo "数据库表创建完成"

# 初始化 admin 管理员（仅首次部署执行）
$PYTHON -c "from database.sql_db.conn import init_rds_data; init_rds_data()"
echo "admin 用户初始化完成（用户名: admin, 密码: admin123）"

# 清除当前终端的环境变量
unset DASHGO_MASTER_KEY
```

## 第五步：创建 systemd 服务文件

### 5.1 主应用服务（Web）

```bash
sudo tee /etc/systemd/system/dashgo.service > /dev/null << 'EOF'
[Unit]
Description=DashGo Web Application
After=network.target mysql.service
Wants=mysql.service

[Service]
Type=exec
User=root
Group=root
WorkingDirectory=/opt/my_project/Plotly-DashGo/src
EnvironmentFile=/etc/dashgo/env
ExecStart=/opt/my_project/Plotly-DashGo/.venv/bin/gunicorn \
    --capture-output \
    --workers 4 \
    --bind 0.0.0.0:8090 \
    --timeout 120 \
    --access-logfile /opt/dashgo/logs/access.log \
    --error-logfile /opt/dashgo/logs/error.log \
    app:server
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

### 5.2 任务调度服务（APScheduler）

```bash
sudo tee /etc/systemd/system/dashgo-scheduler.service > /dev/null << 'EOF'
[Unit]
Description=DashGo Task Scheduler (APScheduler)
After=network.target mysql.service dashgo.service
Wants=mysql.service

[Service]
Type=exec
User=root
Group=root
WorkingDirectory=/opt/my_project/Plotly-DashGo/src
EnvironmentFile=/etc/dashgo/env
ExecStart=/opt/my_project/Plotly-DashGo/.venv/bin/python app_apscheduler.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

### 5.3 创建日志目录

```bash
sudo mkdir -p /opt/dashgo/logs
```

---

## 第六步：启动服务

```bash
# 重新加载 systemd 配置
sudo systemctl daemon-reload

# 启动主应用
sudo systemctl start dashgo

# 启动任务调度（如需使用任务中心模块）
sudo systemctl start dashgo-scheduler

# 设置开机自启
sudo systemctl enable dashgo
sudo systemctl enable dashgo-scheduler
```

---

## 第七步：验证部署

```bash
# 查看主应用状态（应显示 active (running)）
sudo systemctl status dashgo

# 查看调度器状态
sudo systemctl status dashgo-scheduler

# 查看应用日志
sudo journalctl -u dashgo -f --no-pager -n 50

# 测试 HTTP 访问
curl -I http://127.0.0.1:8090/
# 应返回 HTTP 302（重定向到登录页）
```

浏览器访问 `http://你的服务器IP:8090/`，使用默认账号登录：
- 用户名：`admin`
- 密码：`admin123`

> 首次登录后请立即修改默认密码。

## 日常运维命令速查

| 操作 | 命令 |
|------|------|
| 启动应用 | `sudo systemctl start dashgo` |
| 停止应用 | `sudo systemctl stop dashgo` |
| 重启应用 | `sudo systemctl restart dashgo` |
| 查看状态 | `sudo systemctl status dashgo` |
| 查看实时日志 | `sudo journalctl -u dashgo -f` |
| 查看最近 100 行日志 | `sudo journalctl -u dashgo -n 100 --no-pager` |
| 启动调度器 | `sudo systemctl start dashgo-scheduler` |
| 停止调度器 | `sudo systemctl stop dashgo-scheduler` |
| 重启调度器 | `sudo systemctl restart dashgo-scheduler` |
| 取消开机自启 | `sudo systemctl disable dashgo` |

## 更新代码后的操作

```bash
# 1. 停止服务
sudo systemctl stop dashgo
sudo systemctl stop dashgo-scheduler

# 2. 更新代码
cd /opt/dashgo
git pull   # 或重新上传代码

# 3. 如有新依赖，重新安装
source .venv/bin/activate
pip install -r requirements_linux.txt
deactivate

# 4. 如有数据库变更，执行迁移
export DASHGO_MASTER_KEY="DashGo@MasterKey2026!"
cd /opt/dashgo/src
/opt/dashgo/.venv/bin/python -c "from database.sql_db.conn import create_rds_table; create_rds_table()"
unset DASHGO_MASTER_KEY

# 5. 重启服务
sudo systemctl start dashgo
sudo systemctl start dashgo-scheduler
```

## 安全清单

- [ ] 主密钥文件 `/etc/dashgo/env` 权限为 `600`（仅 root 可读）
- [ ] 主密钥没有出现在代码仓库、shell history、日志文件中
- [ ] `dashgo.ini` 中的敏感值均为 `ENC(...)` 格式
- [ ] 已修改默认 admin 密码
- [ ] 数据库用户仅授权了必要的库，未使用 root 账户
- [ ] 如需公网访问，前置 Nginx 反向代理并配置 HTTPS
- [ ] 防火墙仅开放必要端口（如 80/443），不直接暴露 8090
