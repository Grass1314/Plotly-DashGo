# DashGo 数据库配置说明

## 一、支持的数据库类型

- **SQLite**（默认，开发/小规模）
- **MySQL 5.7+**（生产推荐）

---

## 二、配置文件位置与节

配置文件路径：**`src/config/dashgo.ini`**

数据库相关配置在 **`[SqlDbConf]`** 节：

```ini
[SqlDbConf]
# 数据库类型：sqlite 或 mysql
RDB_TYPE = sqlite

# ========== SQLite（当 RDB_TYPE=sqlite 时使用）==========
SQLITE_DB_PATH = ../app.db

# ========== MySQL（当 RDB_TYPE=mysql 时使用）==========
# 敏感值支持 ENC(...) 加密格式，详见「第六节 敏感信息加密」
HOST = 127.0.0.1
PORT = 3306
USER = root
PASSWORD = password
DATABASE = app
# 连接池大小
POOL_SIZE = 5
```

---

## 三、使用 MySQL 时你需要做的

1. **在服务器上安装并启动 MySQL 5.7+**，并确保本机或应用能访问（防火墙、bind-address 等）。

2. **创建数据库**（名称与 `DATABASE` 一致，例如 `app`）：
   ```sql
   CREATE DATABASE app CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

3. **创建有权限的用户**（若不用 root）并授权：
   ```sql
   CREATE USER 'your_user'@'%' IDENTIFIED BY 'your_password';
   GRANT ALL PRIVILEGES ON app.* TO 'your_user'@'%';
   FLUSH PRIVILEGES;
   ```

4. **修改 `src/config/dashgo.ini`** 中 `[SqlDbConf]`：
   - `RDB_TYPE = mysql`
   - `HOST`：MySQL 所在主机（本机填 `127.0.0.1` 或 `localhost`）
   - `PORT`：一般为 `3306`
   - `USER` / `PASSWORD`：上面创建的用户名和密码
   - `DATABASE`：数据库名，如 `app`
   - `POOL_SIZE`：连接池大小，可按并发量调整（默认 5）

5. **初始化表结构和初始数据**（在 `src` 目录下执行）：
   ```bash
   cd /path/to/Plotly-DashGo/src

   # 如果配置文件中使用了 ENC(...) 加密值，需先设置主密钥
   export DASHGO_MASTER_KEY="你的主密钥"

   # 创建所有表（若表已存在会跳过）
   python -c "from database.sql_db.conn import create_rds_table; create_rds_table()"

   # 初始化 admin 用户（仅首次；若已存在可能报错，可忽略或先清空用户表再执行）
   python -c "from database.sql_db.conn import init_rds_data; init_rds_data()"
   ```

6. **默认管理员账号**：用户名 `admin`，密码 `admin123`。上线后请尽快修改。

## 四、数据库表一览（供你建库/排查用）

应用会通过 **Peewee ORM** 自动建表（`create_rds_table()`），你**不需要**手动建表，只要保证 MySQL 里已存在**空库**（如 `app`）即可。

| 表名 | 说明 |
|------|------|
| sys_user | 用户 |
| sys_role | 角色 |
| sys_group | 团队/用户组 |
| sys_user_role | 用户-角色 |
| sys_group_user | 团队-用户 |
| sys_group_role | 团队-角色 |
| sys_role_access_meta | 角色权限元数据 |
| sys_announcement | 公告 |
| oauth2_client | OAuth2 客户端 |
| oauth2_authorization_code | OAuth2 授权码 |
| oauth2_token | OAuth2 令牌 |
| sys_apscheduler_results | 调度任务结果 |
| sys_apscheduler_extract_value | 调度抽取值 |
| sys_apscheduler_running | 任务运行日志 |
| sys_notify_api | 通知 API 配置 |
| sys_listen_api | 监听 API 配置 |
| apscheduler_jobs_active_listen | 主动监听任务配置 |

## 五、连接与代码位置

- **连接与池**：`src/database/sql_db/conn.py`
  - SQLite：直接使用 `SqlDbConf.SQLITE_DB_PATH`
  - MySQL：使用 `ReconnectMixin + PooledMySQLDatabase`，断线重连 + 连接池
- **表实体**：`src/database/sql_db/entity/*.py`
- **DAO**：`src/database/sql_db/dao/*.py`

## 六、敏感信息加密

项目支持对 `dashgo.ini` 中的敏感配置值进行 AES-256-GCM 加密存储。

### 加密格式

加密后的值以 `ENC(...)` 格式写入配置文件，例如：

```ini
[SqlDbConf]
HOST = ENC(HaOUrM+jurDLB2sgQEr8EnTU8g7mXugUeyeaqbmjPGk4qBKjvg6wXrc=)
USER = ENC(lgIe7ilpbQFeakm+Ap6+fJ4ZfvgKIDw0WTtBi/MtYfErtG6sh8HbhPX3PKKRWW4=)
PASSWORD = ENC(ZVWGx6q9k9Lvf11bCozTidrZDIjqWYvf68vR/WRx8Zzj4uY9SMJnOjaM7pg=)
```

### 当前已加密的字段

| 节 | 字段 | 说明 |
|----|------|------|
| `CommonConf` | `ENCRYPT_KEY` | 应用数据加密密钥 |
| `FlaskConf` | `COOKIE_SESSION_SECRET_KEY` | Cookie 会话密钥 |
| `JwtConf` | `JWT_SECRET_KEY` | JWT 签名密钥 |
| `SqlDbConf` | `HOST` | 数据库地址 |
| `SqlDbConf` | `USER` | 数据库用户名 |
| `SqlDbConf` | `PASSWORD` | 数据库密码 |

### 主密钥设置

应用启动时需通过环境变量 `DASHGO_MASTER_KEY` 提供主密钥：

```bash
# 命令行启动
export DASHGO_MASTER_KEY="你的主密钥"
cd src && python app.py

# 生产环境（gunicorn）
export DASHGO_MASTER_KEY="你的主密钥"
cd src && gunicorn --capture-output -w 4 -b 0.0.0.0:8090 app:server
```

若配置文件中存在 `ENC(...)` 值但未设置 `DASHGO_MASTER_KEY`，启动时会报错并提示。

若配置文件中全部为明文值（无 `ENC(...)` 前缀），则不需要设置环境变量，向后兼容。

### 加密工具使用

项目提供了命令行加密工具 `src/config/encrypt_tool.py`：

```bash
cd src

# 交互式加密单个值
python config/encrypt_tool.py

# 一键加密 dashgo.ini 中的所有预定义敏感字段
python config/encrypt_tool.py --encrypt-ini

# 解密验证一个 ENC(...) 值
python config/encrypt_tool.py --decrypt
```

### 生产环境推荐：systemd 服务

将主密钥写入独立的环境变量文件（权限 600，仅 root 可读），通过 systemd 的 `EnvironmentFile` 注入：

```bash
# 创建环境变量文件
sudo mkdir -p /etc/dashgo
echo 'DASHGO_MASTER_KEY=你的主密钥' | sudo tee /etc/dashgo/env
sudo chmod 600 /etc/dashgo/env
```

```ini
# /etc/systemd/system/dashgo.service
[Unit]
Description=DashGo Web Application
After=network.target mysql.service

[Service]
User=www
WorkingDirectory=/你的项目路径/src
EnvironmentFile=/etc/dashgo/env
ExecStart=/你的项目路径/.venv/bin/gunicorn --capture-output -w 4 -b 0.0.0.0:8090 app:server
Restart=always

[Install]
WantedBy=multi-user.target
```

## 七、注意事项

- 生产环境请勿使用默认密码和 JWT/加密相关密钥，请修改后通过加密工具加密并妥善保管主密钥。
- SQLite 的 `SQLITE_DB_PATH` 为相对路径时，依赖「当前工作目录」；按 README 在 `src` 下启动即可。
- 若 `dashgo.ini` 中缺少 `[SqlDbConf]` 或上述任一键，应用启动时可能报错，请对照本文检查配置。
- `dashgo.ini` 文件已包含加密后的敏感值，可以安全地提交到版本控制；但主密钥（`DASHGO_MASTER_KEY`）绝对不要提交到代码仓库。
